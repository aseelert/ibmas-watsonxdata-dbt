Presto / IBM watsonx.data Iceberg Execution Details

All dbt models execute via Presto against an Apache Iceberg table format on IBM watsonx.data. Silver and Gold tables use format = 'PARQUET' with partitioning = ARRAY['month(order_date)'] for time-based query pruning. The gold_category_performance model is a VIEW (no stored data) computed on-demand over gold_daily_sales. bronze models carry ingestion metadata columns: _ingested_at, _ingested_by, _source_file, _ingest_batch_id. The silver layer applies explicit type casts (CAST … AS INTEGER / DATE / TIMESTAMP / DECIMAL), lower/upper/trim normalisation, and WHERE email IS NOT NULL / WHERE order_id IS NOT NULL deduplication filters before writing Parquet. The enriched silver fact silver_sales_enriched is the single source of truth read by all three Gold marts — no raw data is re-joined at Gold level. The datastage-join asset discovered in the project confirms a parallel DataStage pipeline also writes to the same project scope.
